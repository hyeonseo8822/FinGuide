# ELS Experience Redesign

## Core Direction

The current ELS page feels too much like a report.

Users must scroll through large amounts of information, read long explanations, and look at historical examples.

This approach makes it difficult for students to stay engaged and understand how ELS actually works.

The ELS page should be redesigned as an interactive experience where users learn by making choices and seeing the results.

The goal is not to explain a real historical case.

The goal is to help users understand the structure, rewards, and risks of ELS through experience.

---

# Remove Current Structure

Remove:

* Samsung Electronics real stock data
* 2023 case study
* Fixed historical scenarios
* Long report-style explanations
* "Minjun's Choice" story
* ELS checklist section
* OX quiz section from the ELS page

Move OX quizzes to the Quiz tab.

Keep educational content concise and visual.

---

# Replace "Minjun's Choice" With "My Choice"

Instead of showing a character making decisions, the user should become the main character.

Title:

🎮 My Choice

Example:

"You have 10 million won.

Which option would you choose?"

Display two large cards:

🐷 Savings Deposit

Safe and predictable return

🎲 ELS

Higher potential return but additional risk

The user chooses directly.

---

# Interactive Experience Flow

The page should feel like a simple story game.

Step 1

User selects:

🐷 Savings Deposit

or

🎲 ELS

---

Step 2

Generate a fictional company.

Do not use Samsung Electronics.

Examples:

🏢 Future Tech

🏢 AI Electronics

🏢 Dream Semiconductor

🏢 Smart Device

Randomly choose one company.

---

Step 3

Generate random stock price movement.

Each refresh should create a different scenario.

Example:

Starting Price

50,000 KRW

↓

6 Months Later

62,000 KRW

or

41,000 KRW

or

28,000 KRW

Random outcome.

The user should not know the result beforehand.

---

Step 4

Show what happened.

Example:

📈 The stock price stayed above the required level.

or

📉 The stock price dropped below the danger line.

---

Step 5

Display the final result.

Example:

🎉 Success

ELS conditions were met.

You received the promised return.

or

😢 Loss

The stock price fell too much.

A principal loss occurred.

---

# Compare With Savings Deposit

After showing the ELS result, immediately show:

"What would have happened if you chose a savings deposit?"

Example:

🐷 Savings Deposit

10,000,000 KRW

↓

10,400,000 KRW

Guaranteed result

Compare it with the ELS result.

This helps users understand both reward and risk.

---

# Educational Focus

Do not simply tell users that ELS is dangerous.

Let users experience why it can be risky.

The lesson should come from the outcome.

Example:

🎉 Good Scenario

ELS earns more than savings.

Users learn:

"Higher returns are possible."

Example:

😢 Bad Scenario

ELS loses money.

Users learn:

"Higher returns come with higher risks."

---

# Result Screen

The result screen should answer:

* Why did this happen?
* Was the ELS condition met?
* What would have happened with a savings deposit?
* What lesson should the user learn?

Keep explanations short.

Use icons and visuals.

Avoid long paragraphs.

---

# Sources Section

The References section should only contain sources that were actually used.

Do not list organizations that were not referenced.

Example:

Sources & References

This page was created using information from:

* Financial Supervisory Service
* Korea Financial Investment Association
* Other sources actually used in the project

Only display real references.

---

# Design Requirements

The page should feel like:

* A story
* A simulation
* A simple game
* An educational experience

Not:

* A financial report
* A textbook
* A long article

Use:

* Large cards
* Animations
* Icons
* Visual comparisons
* Interactive choices

The user should learn ELS by experiencing outcomes rather than reading long explanations.
